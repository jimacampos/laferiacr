# Spec-driven development (Spec Kit)

This repo uses **Spec Kit** for spec-driven development, installed under `laferiaspecs/`.

- **Constitution (binding principles):** `laferiaspecs/.specify/memory/constitution.md` (v1.0.0 — code
  quality, testing standards, UX consistency, performance). Follow it in every change.
- **Feature specs:** `laferiaspecs/specs/<NNN-feature>/` hold `spec.md` (what/why) and `plan.md` (how)
  for forward-looking roadmap phases (currently 4.5, 6, 7, 8). Read a feature's `spec.md` before
  building/changing it, and keep the spec in sync. Index: `laferiaspecs/specs/README.md`.
- **Reference docs:** `docs/` is the source of truth for shipped behavior, ADRs, architecture,
  personas, accessibility, and glossary. Specs link back to `docs/` rather than duplicating it.
- **Commands:** the `/speckit.*` prompts/agents live under `laferiaspecs/.github/prompts/` and
  `laferiaspecs/.github/agents/`. Flow: `/speckit.constitution` → `/speckit.specify` →
  `/speckit.clarify` → `/speckit.plan` → `/speckit.tasks` → `/speckit.implement`. `tasks.md` is
  generated by `/speckit.tasks`, not written by hand.

See `.github/copilot-instructions.md` for the full repo-wide Copilot guidance.
